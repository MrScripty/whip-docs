# src-tauri/src/graph

## Purpose
This directory owns versioned architecture graph contracts, stable IDs,
directory/file graph building, and normalization from analyzer facts into
frontend-readable snapshots.

## Contents
| File/Folder | Description |
|-------------|-------------|
| `mod.rs` | Versioned analyzer graph DTOs, V0 directory/file graph DTOs, stable ID helpers, and deterministic directory graph building. |

## Problem
Frontend graph rendering and backend extraction need a stable contract before
parallel work can proceed.

## Constraints
- Graph snapshots are backend-owned and versioned.
- Node and edge IDs must be stable across equivalent analysis runs.
- Low-confidence or skipped data is represented as diagnostics.

## Decision
Define graph DTOs and normalization here before analyzer and UI implementation
depend on them.

For V0, build the repository directory/file graph here too. The frontend
consumes the resulting snapshot as projection data and does not derive graph
facts from raw filesystem paths.

## Alternatives Rejected
- Reuse the old `rust-doc-tool` JSON shape: rejected because it only models
  files and module/import edges.
- Let the frontend infer graph structure from files: rejected because semantic
  analysis belongs in Rust.

## Invariants
- Snapshot schema version is present.
- Graph IDs are deterministic for stable source facts.
- Source ranges are relative to validated source roots.
- Directory graph node paths are relative to the validated repository root.

## Revisit Triggers
- Snapshots become too large for direct Tauri IPC.
- Rust macro expansion support changes edge confidence semantics.
- Non-Rust language support is added.

## Dependencies
**Internal:** analyzer facts, source path validation, command DTOs.
**External:** serde.

## Related ADRs
- `docs/adr/ADR-001-tauri-rust-analyzer-graph-contracts.md`: records the
  backend-owned graph snapshot and DTO contract decision.

## Usage Examples
```rust
let repo = ValidatedRepoPath::parse_existing_cargo_repo(raw_path)?;
let snapshot = DirectoryGraphBuilder::build(&repo)?;
```

## API Consumer Contract
- Inputs: analyzer facts and validated source metadata.
- Outputs: versioned graph snapshots, directory graph snapshots, and diagnostics.
- Lifecycle: analyzer snapshots are generated by explicit analysis jobs; V0
  directory graph snapshots are generated from validated repository paths.
- Errors: incomplete extraction is represented through diagnostics where useful.
- Compatibility: breaking graph changes require synchronized frontend updates.

## Structured Producer Contract
- Stable fields: schema version, node IDs, edge IDs, source ranges, diagnostics.
- Defaults: absent optional provenance means exact backend-owned extraction.
- Enum semantics: node and edge kinds are behavior-bearing labels.
- Compatibility: append fields where possible; breaking changes require ADR
  updates.
- Regeneration or migration: saved snapshots must include schema version before
  persistence is introduced.
